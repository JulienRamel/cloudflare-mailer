# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please send an email to **julien@ramel.io**.
You should receive a response within 48 hours. If you do not, please follow up to ensure
the original message was received.

Please include as much of the following information as possible to help understand and
reproduce the issue:

- Type of issue (e.g. API token exposure, input validation bypass)
- Full path of the source file(s) related to the issue
- Steps to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact of the issue

This information will help triage your report more quickly.
