# Changelog

## [Unreleased]

### Added
- Initial release: `CloudflareApiTransport` for Cloudflare Email Service REST API
- `CloudflareTransportFactory` with DSN support (`cloudflare+api://ACCOUNT_ID:API_TOKEN@default`)
- `CloudflareBounceEvent` dispatched when permanent bounces are reported in the API response
- Support for CC, BCC, reply-to, attachments, inline images and custom headers
- Symfony 8.0 support (`symfony/mailer: ^7.2|^8.0`)

### Fixed
- Inline image Content-IDs generated by Symfony (e.g. `@images/logo.png`) are sanitized to plain
  basenames before submission to the Cloudflare API, which rejects `@` and path separators in CIDs.
  The HTML body is rewritten accordingly so `cid:` references remain valid.
